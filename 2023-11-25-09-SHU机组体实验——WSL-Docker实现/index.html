SHU机组体实验——WSL+Docker实现

发表于2023-11-25|更新于2023-11-25|Learning
|字数总计:829|阅读时长:2分钟|阅读量:

在Windows11上以Docker部署CentOS容器作为虚拟机进行试验。

前置：安装好WSL和Docker Desktop。

虚拟容器的创建

通过Docker创建的虚拟容器是自定义的，可以根据需求进行安排。因此可以快速轻量的完成部署，且避免了传统虚拟机的性能损耗。通过我的Dockerfile创建的CentOS，无需配置网卡，开箱即用。

Docker相关的概念不做多余介绍。

其他的就不做多余描述，请参考网络上相关的教程即可。
下面是Dockerfile。

从dockerfile创建镜像。此时指令目录为dockerfile所在目录。
docker build -t mycentos:dev .

构建好镜像后创建并启动容器。注意将容器的80端口映射到主机的45678端口(此端口不指定，大于1200即可)，以便后续SSH连接。指令如下。请查阅docker --help查看相关参数含义。
`docker run -p 45678:22 -it -d –name=my_centos_1 mycentos:dev

以bash进入容器。
docker exec -it my_centos_1 /bin/bash

以上是基本操作，根据实验内容创建第二个容器也是相似的操作。

宿主机与虚拟机通信

WSL与CentOS容器以SSH相互通信

首先介绍在任意的WSL实例中进行通行。
！我并未测试过多个WSL实例会出现什么情况，读者可以进行实验，希望得到反馈！

WSL登录CentOS容器

在容器中，查找到IP。
切换到WSL，以SSH连接。举例如下。
ssh root@172.17.0.2
输入设置的密码即可。

从CentOS容器登录到WSL

在WSL中通过指令查看IP。
切换到容器。命令如下。
ssh <用户名>@192.168.158.4

PWSh与CentOS容器以SSH相互通信

从容器或WSL登录宿主机

首先在WSL中查看主机的IP。
cat /etc/resolv.conf
后通过SSH连接即可。在WSL和CentOS容器中都是相同的操作。这样可以登录到宿主机上。

从宿主机登录到WSL

在WSL查看IP。
后在Windows输入以下命令即可。
ssh <user>@<your_IP>

从宿主机或WSL登录到容器

SSH需要开放22端口，如果没有在一开始设置好22端口的话，将无法使用SSH。
如果从WSL进行登录的话，IP可以是localhost，也可以是Docker分配到的IP，但都必须加入端口映射。
如果从Windows进行登录的话，IP只能是localhost。

容器间相互通信

我构建的镜像默认的网络设置就是桥接(所有的容器连接到Docker虚拟机上的网卡，再连接到WSL，WSL以默认NAT的方式设置)。因此可以直接通过SSH在各容器间相互通信。

自此，通信内容全部打通。

总结

SSH默认的监听端口是22。在宿主机上做端口映射是将容器的端口映射到主机上的端口，如果在容器中SSH监听的端口和所映射的端口不一致的话，SSH无法正常工作。
WSL和Docker采用的虚拟化技术十分先进，建议使用。
